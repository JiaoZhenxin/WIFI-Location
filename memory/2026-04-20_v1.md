## 当前真实边界

当前项目已经不是 demo 状态，而是一个“已建仓、已提交、已推远端”的开发基线。

但仍有几个现实边界：

1. 当前算法框架已经具备抗漂移设计，但还没有用真实工厂指纹库完成精度验证。
2. 目标精度 1 到 2 米，仍然取决于现场采集质量、AP 布局和设备一致性。
3. 当前版本更像“可持续开发基线”，还不是最终现场交付版本。
4. 文档目录当前实际名称是 `desgin`，不是 `design`。

## 当前遗留项

### 高优先级

1. 建立正式的现场指纹采集流程。
2. 支持指纹库管理：
   导入、导出、覆盖、删除、版本管理。
3. 用真实工厂环境做静止点、路径点、遮挡场景验证。

### 中优先级

1. 引入 IMU / PDR 融合，进一步抑制轨迹漂移。
2. 接入真实厂区地图底图和区域边界约束。
3. 增强定位置信度表达，不可靠时不要强制出点。

### 低优先级

1. 优化产品化 UI。
2. 增加更完整的调试面板和定位回放。
3. 调整文档目录名称：
   `desgin -> design`

## 后续建议执行顺序

建议后续回来继续开发时按以下顺序推进：

1. 先做正式采集模式与指纹库治理。
2. 再做真实现场测试，确认当前 base 的精度和漂移边界。
3. 如果现场测试显示动态漂移仍明显，再做 IMU / PDR 融合。
4. 之后再做底图、区域约束和产品化体验提升。

## 决策建议

如果后续时间有限，只能选一个方向先推进：

### 方案 A：优先做采集体系

适合目标：

1. 尽快验证项目是否具备实际落地价值。
2. 尽快找出精度问题到底来自数据还是算法。

结论：

- 这是最推荐的下一步。

### 方案 B：优先做 IMU / PDR 融合

适合目标：

1. 当前已经有较好的指纹数据。
2. 主要痛点已经从“匹配错误”转成“轨迹跳动”。

结论：

- 适合作为第二阶段，不建议跳过采集体系直接做。

### 方案 C：优先做地图与界面

适合目标：

1. 优先提升展示效果。
2. 做对外演示或内部汇报。

结论：

- 展示价值高，但对精度本身帮助不如前两个方向。
