# Factory WiFi Locator 软件设计文档

## 1. 文档目的

本文档定义 `wifi-positioning-app` 的 Android 实现方案，用于在工厂环境中基于 WiFi 指纹实现室内定位，并重点解决原 demo 在复杂多径环境下存在的位置漂移问题。本文档覆盖系统目标、模块划分、关键流程、数据结构、定位算法、数学理论与工程约束。

## 2. 设计目标

### 2.1 功能目标

1. 基于 Android 手机扫描周边 AP 的 RSSI，输出当前平面坐标。
2. 支持本地指纹库加载、增量采集、覆盖工厂现场标定。
3. 在 UI 上展示参考点、原始估计点、稳定估计点和候选点。
4. 支持连续扫描定位与现场指纹点采集。

### 2.2 性能目标

1. 静止状态下显著降低点位抖动和来回跳变。
2. 在完成足够密度的现场标定后，定位目标精度为 1 到 2 米。
3. 扫描和定位链路可在中端 Android 设备上实时运行。

### 2.3 约束说明

1. 1 到 2 米精度无法只靠代码保证，必须依赖现场 AP 布局、参考点密度、标定质量和手机型号一致性。
2. Android 的 `startScan()` 受系统限流，工厂部署时建议使用企业设备、关闭节电策略，并在现场做扫描周期验证。
3. 工厂金属设备、多径反射、人员遮挡会导致 RSSI 非高斯波动，因此算法必须具备鲁棒性和时序约束。

## 3. demo 问题分析

原 demo 的核心思想是 WiFi 指纹匹配，但从结构上看更接近“短时间 RSSI 向量直接匹配后立即出点”。这种方法在工厂场景下会出现以下问题：

1. 单帧 RSSI 方差大，瞬时扫描结果会因为遮挡或反射出现 5 到 15 dBm 的波动。
2. 指纹匹配直接输出坐标，缺少轨迹连续性约束，导致相邻时刻在多个近邻参考点之间跳变。
3. 对 AP 缺失、新出现 AP、异常值的处理较弱，在工厂环境里很容易把偶发强信号误判为位置跃迁。
4. 没有把测量方差、重叠率、候选点空间离散程度转成“可信度”，因此系统无法对低可信观测降权。

## 4. 系统架构

### 4.1 模块划分

1. `MainActivity`
   负责权限申请、扫描控制、定位结果展示、指纹点保存和恢复示例库。
2. `WifiScanner`
   负责 WiFi 扫描循环、广播接收、扫描摘要生成。
3. `FingerprintStore`
   负责从 assets 或内部存储加载、保存指纹库。
4. `PositioningEngine`
   负责稳定观测构建、指纹相似度计算、候选点加权融合。
5. `TemporalPositionFilter`
   负责基于状态空间模型抑制抖动与异常跃迁。
6. `FloorMapView`
   负责将参考点、原始点、滤波点绘制到平面图上。

### 4.2 运行流程

1. App 启动后加载本地指纹库。
2. 用户点击开始扫描。
3. `WifiScanner` 周期性触发 WiFi 扫描并接收结果。
4. 系统维护最近 `N=6` 帧扫描窗口。
5. `PositioningEngine` 将扫描窗口融合为稳定观测向量。
6. 稳定观测与指纹库逐点打分，取 top-k 候选点。
7. 对 top-k 候选点做加权质心求解，得到原始坐标。
8. `TemporalPositionFilter` 对原始坐标做卡尔曼更新与门限剔除，输出稳定坐标。
9. UI 同时展示原始点、稳定点和候选点分数。

## 5. 数据设计

### 5.1 指纹库文件

文件路径：

- `app/src/main/assets/fingerprint_seed.json`
- 运行时副本：`files/fingerprint_db.json`

文件结构：

```json
{
  "siteWidthMeters": 30.0,
  "siteHeightMeters": 18.0,
  "fingerprints": [
    {
      "id": "P01",
      "x": 3.0,
      "y": 3.0,
      "samples": {
        "BSSID": { "mean": -46.0, "stdDev": 2.5, "count": 16 }
      }
    }
  ]
}
```

### 5.2 数据对象

1. `AccessPointStats`
   表示某个参考点处某个 BSSID 的均值、标准差和采样数量。
2. `FingerprintPoint`
   表示单个参考点，包括点位名称、坐标和该点的 AP 统计特征。
3. `FingerprintDatabase`
   表示整个场地的指纹库和地图尺寸。
4. `PositionEstimate`
   包含原始估计坐标、滤波后坐标、置信半径和候选点列表。

## 6. 抗漂移定位算法设计

### 6.1 总体思路

本方案不是单一算法，而是一条抗漂移链路：

1. 多帧观测稳态化。
2. 鲁棒指纹匹配。
3. 候选点加权融合。
4. 时序滤波和异常门限。

核心目标是把“瞬时 RSSI 抖动”变成“低频位置变化”，再把“多个候选点的离散跳跃”变成“物理上连续的轨迹”。

### 6.2 多帧稳态观测

设第 `t` 次扫描得到的 WiFi 测量为：

\[
\mathbf{s}_t = \{(b_1, r_{t,1}), (b_2, r_{t,2}), \dots\}
\]

其中 `b_i` 是 BSSID，`r_{t,i}` 是 RSSI。

系统保留最近 `N=6` 帧扫描窗口：

\[
\mathcal{W}_t = \{\mathbf{s}_{t-N+1}, \dots, \mathbf{s}_t\}
\]

对每个 BSSID 单独聚合，记其 RSSI 序列为：

\[
R_b = \{r_1, r_2, \dots, r_m\}
\]

先计算中位数：

\[
\tilde{r}_b = \mathrm{median}(R_b)
\]

再计算中位绝对偏差：

\[
\mathrm{MAD}_b = \mathrm{median}(|r_i - \tilde{r}_b|)
\]

然后剔除离群值：

\[
|r_i - \tilde{r}_b| \le 2.5 \cdot \max(1, \mathrm{MAD}_b)
\]

保留下来的值做均值，形成稳定观测：

\[
\hat{r}_b = \frac{1}{m'} \sum_{i=1}^{m'} r_i
\]

这一过程对应代码中的 `buildStableObservation()`。其作用是用鲁棒统计替代单帧 RSSI，显著降低金属反射、人员遮挡带来的尖峰噪声。

### 6.3 指纹距离建模

设参考点 `p_i` 的指纹特征为：

\[
F_i = \{(b, \mu_{i,b}, \sigma_{i,b})\}
\]

其中 `\mu_{i,b}` 是该点该 BSSID 的均值 RSSI，`\sigma_{i,b}` 是标准差。

实时观测为：

\[
O = \{(b, \hat{r}_b)\}
\]

对某个参考点 `p_i`，若观测和指纹同时包含某个 BSSID，则计算标准化残差：

\[
z_{i,b} = \frac{\hat{r}_b - \mu_{i,b}}{\max(\sigma_{i,b}, \sigma_{\min})}
\]

这里设置：

\[
\sigma_{\min} = 2.0
\]

避免标准差过小导致某个 AP 权重异常放大。

参考点总距离定义为：

\[
D_i = \sum_{b \in O \cap F_i} z_{i,b}^2 + \lambda_m \cdot M_i + \lambda_n \cdot N_i
\]

其中：

1. `M_i` 表示参考点存在但实时观测缺失的 AP 数量。
2. `N_i` 表示实时观测出现但参考点未记录的 AP 数量。
3. `\lambda_m` 是缺失惩罚。
4. `\lambda_n` 是新增惩罚。

当前实现中：

\[
\lambda_m = 3.2,\quad \lambda_n = 0.8
\]

这等价于把 AP 缺失视为更严重的不匹配，把偶发出现的新 AP 视为较轻的扰动。

### 6.4 重叠率约束

只看距离还不够，因为某些参考点可能只和当前观测重叠了很少几个 AP。定义重叠率：

\[
\rho_i = \frac{|O \cap F_i|}{|F_i|}
\]

若：

\[
\rho_i < 0.25
\]

则直接丢弃该参考点。这样可以防止“少量 AP 恰好相似”导致的误命中。

### 6.5 候选点打分

为了让距离小、重叠率高的参考点拥有更高权重，定义候选点分数：

\[
w_i' = \exp\left(-\frac{\sqrt{D_i}}{\tau}\right)\cdot \rho_i^{\gamma}
\]

当前实现参数：

\[
\tau = 1.6,\quad \gamma = 1.5
\]

这是一种软指数衰减。相比直接使用 `1/D_i`，它对近邻更稳定，对远点衰减更快。

### 6.6 Top-k 加权质心

对分数最高的 `k=4` 个参考点做加权质心：

\[
\hat{x}_{raw} = \frac{\sum_{i=1}^{k} w_i' x_i}{\sum_{i=1}^{k} w_i'}
\]

\[
\hat{y}_{raw} = \frac{\sum_{i=1}^{k} w_i' y_i}{\sum_{i=1}^{k} w_i'}
\]

这是对传统 WKNN 的改造。传统 WKNN 通常只依据欧氏距离倒数加权，而这里的权重已经融合了 RSSI 标准化残差和 AP 重叠率，因此更适合工厂环境中的指纹匹配。

### 6.7 置信半径估计

用候选点围绕原始估计点的离散程度估计可信半径：

\[
r_c = \frac{\sum_{i=1}^{k} w_i' \cdot \sqrt{(x_i-\hat{x}_{raw})^2 + (y_i-\hat{y}_{raw})^2}}{\sum_{i=1}^{k} w_i'}
\]

候选点越集中，`r_c` 越小；候选点越分散，说明当前环境歧义越大。

### 6.8 时序滤波

为了抑制位置跳变，系统使用二维常速度卡尔曼滤波器。状态向量定义为：

\[
\mathbf{x}_k = [x_k,\ y_k,\ v_{x,k},\ v_{y,k}]^T
\]

状态转移矩阵：

\[
\mathbf{F} =
\begin{bmatrix}
1 & 0 & \Delta t & 0 \\
0 & 1 & 0 & \Delta t \\
0 & 0 & 1 & 0 \\
0 & 0 & 0 & 1
\end{bmatrix}
\]

观测矩阵：

\[
\mathbf{H} =
\begin{bmatrix}
1 & 0 & 0 & 0 \\
0 & 1 & 0 & 0
\end{bmatrix}
\]

观测向量：

\[
\mathbf{z}_k = [\hat{x}_{raw},\ \hat{y}_{raw}]^T
\]

预测阶段：

\[
\mathbf{x}_{k|k-1} = \mathbf{F}\mathbf{x}_{k-1|k-1}
\]

\[
\mathbf{P}_{k|k-1} = \mathbf{F}\mathbf{P}_{k-1|k-1}\mathbf{F}^T + \mathbf{Q}
\]

更新阶段：

\[
\mathbf{K}_k = \mathbf{P}_{k|k-1}\mathbf{H}^T (\mathbf{H}\mathbf{P}_{k|k-1}\mathbf{H}^T + \mathbf{R})^{-1}
\]

\[
\mathbf{x}_{k|k} = \mathbf{x}_{k|k-1} + \mathbf{K}_k(\mathbf{z}_k - \mathbf{H}\mathbf{x}_{k|k-1})
\]

\[
\mathbf{P}_{k|k} = (\mathbf{I}-\mathbf{K}_k\mathbf{H})\mathbf{P}_{k|k-1}
\]

其中观测噪声协方差 `R` 根据置信半径自适应设置，意味着候选点越分散，卡尔曼滤波越不相信当前测量。

### 6.9 马氏距离门限

为了防止异常观测直接把滤波状态拉偏，引入创新的马氏距离：

\[
d_M^2 = (\mathbf{z}_k - \hat{\mathbf{z}}_k)^T \mathbf{S}_k^{-1} (\mathbf{z}_k - \hat{\mathbf{z}}_k)
\]

若：

\[
d_M^2 > 9.21
\]

则拒绝本次测量更新。`9.21` 对应二维卡方分布 99% 置信门限。直观上，这一步在做“统计意义上的异常跳点剔除”。

### 6.10 速度约束

滤波更新后，对速度模长做约束：

\[
\sqrt{v_x^2 + v_y^2} \le 1.2\ \text{m/s}
\]

这相当于把工厂步行速度显式写进系统，避免瞬时观测让位置在一秒内跨越数米。

## 7. 精度达成条件

若要把精度稳定压到 1 到 2 米，建议满足以下工程条件：

1. 工厂内目标区域至少可见 4 到 6 个稳定 AP。
2. 参考点网格间距建议 1.5 到 2.0 米。
3. 每个参考点采集 15 到 30 帧扫描，并在不同时间段重复采样。
4. 采集设备型号应与运行设备尽量一致。
5. 对金属货架通道、墙角、机台密集区单独加密参考点。
6. 指纹库应按区域、时段或设备型号分层管理，避免单一模型覆盖差异过大的环境。

## 8. UI 设计

1. 上半部分为地图视图。
   蓝点表示参考点，橙点表示原始估计，绿点表示稳定估计。
2. 下半部分为控制与调试面板。
   包括扫描控制、候选点分数、采集点位录入和扫描窗口摘要。

## 9. 测试设计

### 9.1 单元测试

当前已提供 `PositioningEngineTest`，验证稳定观测窗口能抑制异常 RSSI。

### 9.2 现场测试

建议至少做以下验证：

1. 静止测试。
   人员静止 60 秒，记录稳定点漂移半径。
2. 相邻点切换测试。
   在两点之间往返行走，观察是否出现跨区域跳点。
3. 通道测试。
   沿工位通道匀速行走，检查轨迹连续性。
4. 遮挡测试。
   在设备、人流和推车经过时统计定位误差增量。

## 10. 后续增强方向

1. 引入 IMU 步态约束，把 PDR 与 WiFi 指纹融合。
2. 支持楼层切换和区域分库。
3. 增加指纹点编辑、删除、导入导出。
4. 引入自适应 AP 选择，去掉不稳定热点和移动热点。
5. 对不同手机型号做 RSSI 偏置标定。

## 11. 当前实现与文档对应关系

1. 稳态观测：`PositioningEngine.buildStableObservation()`
2. 指纹评分：`PositioningEngine.estimate()`
3. 时序滤波：`TemporalPositionFilter.update()`
4. 指纹库持久化：`FingerprintStore`
5. 实时扫描：`WifiScanner`
6. 主界面联动：`MainActivity`

## 12. 结论

本方案相较原 demo 的改进重点不是“再换一个 KNN 参数”，而是把定位流程变成“鲁棒统计 + 空间匹配 + 时序约束”的组合系统。只有这样，才能在工厂这种多径严重、遮挡频繁、RSSI 波动明显的场景里实质性降低位置漂移。
