## 2. 先确认我是否真的理解了用户目标

开始动手前，必须先确认以下问题：

1. 用户要我“分析”还是“直接改代码”？
2. 用户要的是“修 bug”还是“做新功能”？
3. 用户关注的是“准确性”还是“可用性”还是“两者都要”？
4. 用户有没有指定目录、文件位置、命名方式或交付格式？
5. 用户有没有说“这会重复发生”，意味着应该顺便沉淀 skill、模板或规范？

如果以上任一项不清楚，先从现有上下文中推断，再决定是否需要补问。

## 3. 先确认项目上下文

接手任务时至少检查以下内容：

1. 当前仓库位置
2. 当前主工程目录
3. 当前文档目录
4. 当前 memory 快照目录
5. 当前 AI 工作手册目录
6. 最近一次快照或复盘是否存在

本项目建议默认先看：

1. `/home/nio/workspace/wifi_location/memory`
2. `/home/nio/workspace/wifi_location/desgin`
3. `/home/nio/workspace/wifi_location/AI 工作手册`
4. `/home/nio/workspace/wifi_location/wifi-positioning-app`

## 4. 代码任务启动前检查

如果这是代码任务，开始前必须确认：

1. 当前改动范围在哪个模块
2. 相关文件有哪些
3. 是否已有未提交改动
4. 是否已有类似文档或约束
5. 是否需要兼容已有构建链
6. 是否需要保留用户现有文件结构

至少先执行或等价确认：

1. `git status`
2. 相关文件阅读
3. 目标路径确认

## 5. Android 任务专项检查

如果任务涉及 Android 前台 app，开始前必须额外检查：

1. 是否涉及按钮点击路径
2. 是否涉及 Activity / Fragment 生命周期
3. 是否涉及扫描、传感器、网络或定时回调
4. 是否涉及文件 IO、JSON、数据库
5. 是否涉及频繁 UI 刷新
6. 是否可能阻塞主线程

### Android 红线提醒

默认禁止直接把以下逻辑放在主线程：

1. 文件读写
2. JSON 解析和序列化
3. 大量排序与聚合
4. 扫描结果处理
5. 定位算法计算
6. 大量文本拼接

如果涉及这些内容，默认先设计后台执行方案。

## 6. 性能与可用性检查

开始实现前先问自己：

1. 用户说的问题是不是“功能错了”，还是“功能虽然对但没法用”？
2. 当前任务是否会影响响应速度？
3. 当前任务是否会引入新的卡顿点？
4. 是否需要防重复点击？
5. 是否需要降频刷新？
6. 是否需要后台线程、锁、快照或版本号？

如果这是交互类任务，不允许只关注功能正确而忽略流畅性。

## 7. 文档与沉淀检查

开始前先判断这次任务是否顺带需要沉淀：

1. 是否需要更新设计文档
2. 是否需要更新开发思路
3. 是否需要写新的 memory 快照
4. 是否需要补 AI 工作手册复盘
5. 是否需要新增 skill、模板或规范

如果任务具有重复性，优先考虑把流程沉淀下来，而不是只完成一次性输出。

## 8. 交付物检查

开始前明确本次最终交付需要什么：

1. 代码
2. 文档
3. APK
4. 测试结果
5. 快照
6. 提交记录
7. GitHub 推送

如果没有提前明确，后面容易遗漏“用户认为理所当然”的内容。

## 9. 执行策略检查

正式动手前，先决定这次要采用哪种策略：

### 策略 A：先读后改

适用情况：

1. 不熟悉代码
2. 风险高
3. 可能影响构建或主流程

### 策略 B：先落骨架再补细节

适用情况：

1. 任务较大
2. 需要多文件联动
3. 需要先建立结构再继续填充

### 策略 C：先修最危险问题

适用情况：

1. 当前问题导致“根本不可用”
2. 用户反馈强烈
3. 需要先止血

如果是 Android 卡顿、崩溃、主流程不可用，优先用策略 C。

## 10. 提交与版本管理检查

如果本次任务会形成代码提交，开始前先确认：

1. 是否要提交到 git
2. 是否要推到 GitHub
3. 提交信息应遵循什么规范
4. 本次改动是否应该单独成一个 commit

本项目默认遵循：

- `/home/nio/workspace/wifi_location/desgin/代码提交规范.md`

## 11. 启动前最终问答

开始执行前，必须对下面问题给出明确答案：

1. 我现在最先该做的第一步是什么？
2. 哪个部分最可能出风险？
3. 哪个部分最容易被忽略但会影响可用性？
4. 本次交付完成后，需要不要补文档或快照？
5. 我是否会再次犯已经在 `AI 工作手册` 里复盘过的问题？

如果第 5 个问题答案不是“不会”，就先回头看：

1. `/home/nio/workspace/wifi_location/AI 工作手册/AI开发红线清单.md`
2. `/home/nio/workspace/wifi_location/AI 工作手册/2026-04-21_Android首版卡顿问题复盘.md`
